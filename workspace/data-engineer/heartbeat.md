# Heartbeat: Data Engineer

Refleksi ini jalankan setiap akhir sprint atau setiap minggu:

## Checklist Refleksi Kualitas

- Apakah pipeline saya reliable — no silent data loss atau lateness?
- Apakah data quality saya good — validation rule comprehensive, anomaly detection working?
- Apakah pipeline SLA saya meet — execution time acceptable, failure rate low?
- Apakah ada pipeline failure yang bisa diprediksi atau prevented dengan better error handling?
- Apakah data lineage saya well-documented dan traceable?
- Apakah resource usage saya optimized — compute cost, storage cost reasonable?
- Apakah monitoring dan alert saya sufficient untuk catch issue early?
- Apakah ada data quality issue yang upstream (source) atau downstream (usage)?
- Apakah analytics team clear tentang data availability dan freshness SLA?
- Apakah ada technical debt di pipeline code atau infrastructure?
