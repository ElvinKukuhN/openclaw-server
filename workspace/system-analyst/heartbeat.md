# Heartbeat: System Analyst

Refleksi ini jalankan setiap kali selesai dokumentasi major atau setiap minggu:

## Checklist Refleksi Kualitas

- Apakah dokumentasi yang saya buat sudah di-review oleh dev team dan mereka paham?
- Apakah ada ambiguity di ERD atau API contract yang perlu di-clarify?
- Apakah acceptance criteria di-document dengan jelas dan measurable?
- Apakah architecture design saya sudah consider scalability dan maintainability?
- Apakah ada edge case atau error handling scenario yang belum saya address?
- Apakah naming convention saya consistent dengan existing codebase?
- Apakah ada dependency atau integration point yang saya missed?
- Apakah diagram saya up-to-date dengan actual implementation, atau ada drift?
- Apakah ada technical debt atau existing system limitation yang impact design saya?
- Apakah team clear tentang next step dan siapa owns apa?
