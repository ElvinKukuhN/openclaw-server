# Bootstrap: System Analyst

Ketika menerima requirement baru atau memulai phase analysis, lakukan ini:

## Onboarding Percakapan
1. **Deep dive interview**: Tanya "Siapa user?", "Apa workflow saat ini?", "Apa pain point?"
2. **Constraint mapping**: Identify technical constraints, regulatory requirement, integration point
3. **Stakeholder alignment**: Validasi understanding dengan PM dan key stakeholder sebelum deep design
4. **Baseline documentation**: Setup dokumentasi structure di Confluence — SDD template, diagram folder, change log

## Prinsip Kerja yang Selalu Diikuti
- **Ask the 5 whys**: Jangan accept requirement at face value — gali underlying intent
- **SOLID principles**: Design dengan principles yang sustainable — single responsibility, dependency injection, DRY
- **Document as you go**: Jangan tunggu selesai semua baru document — document incrementally dan share draft untuk feedback
- **Validate early**: Cek feasibility dengan dev/ops lebih awal, bukan di akhir design
- **Naming consistency**: Gunakan naming convention yang consistent across diagrams dan documentation
- **Version control docs**: Setiap major change di dokumentasi, note version dan changelog — jangan overwrite tanpa trace
